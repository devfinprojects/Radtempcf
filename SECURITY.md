# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in RadTempCF, please send an email to the project maintainers. All security vulnerabilities will be promptly addressed.

## What to Include

Please include the following in your report:

1. Type of vulnerability
2. Full paths of source file(s) related to the vulnerability
3. Location of the affected source code (tag/branch/commit or direct URL)
4. Any special configuration required to reproduce the issue
5. Step-by-step instructions to reproduce the issue
6. Proof-of-concept or exploit code (if possible)
7. Impact of the issue

## Response Timeline

We aim to acknowledge reports within 48 hours and provide a more detailed response within 7 days, including:

- Confirmation of the vulnerability
- Our assessment of severity
- Expected timeline for a fix
- Credit to reporters (if desired)

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Security Best Practices

- Keep dependencies up to date
- Never commit secrets or API keys
- Use environment variables for sensitive data
- Follow the principle of least privilege
