# Contributor Tiers

This document outlines the different contributor tiers in the RadTempCF project and the benefits associated with each level.

## Overview

RadTempCF recognizes contributors at multiple levels, from first-time contributors to project maintainers. Each tier reflects increasing commitment and contribution to the project.

## Tier Levels

### Tier 1: Contributor

**Definition**: Anyone who contributes to the project through code, documentation, templates, or other means.

**Requirements**:
- At least 1 merged PR or accepted template
- Follows contribution guidelines

**Recognition**:
- Listed in the project's contributor list (All-Contributors)
- Mention in release notes for their contribution

**Access**:
- GitHub Discussions access
- Can submit PRs and issues

---

### Tier 2: Regular Contributor

**Definition**: Active contributors who have demonstrated consistent engagement with the project.

**Requirements**:
- 5+ merged PRs or 10+ accepted templates
- Active participation in code review
- Demonstrated understanding of project standards

**Recognition**:
- Profile highlight on contributors page
- Invitation to project Discord (when available)
- Direct mention in community communications

**Access**:
- Triage permissions on GitHub
- Can label issues
- Participate in GitHub Projects

**Responsibilities**:
- Help triage incoming issues
- Review simple PRs
- Mentor new contributors

---

### Tier 3: Committer

**Definition**: Trusted contributors with commit access and increased responsibility.

**Requirements**:
- 20+ merged PRs or 50+ accepted templates
- 6+ months of active contribution
- Demonstrated code review participation
- Sponsor from existing committer

**Recognition**:
- Committer profile page
- Speaking opportunities at conferences
- Co-authorship on academic papers

**Access**:
- Commit access to repository
- Can merge PRs (with approval)
- Access to CI/CD settings

**Responsibilities**:
- Review PRs
- Help maintain documentation
- Participate in design discussions
- Triage issues

---

### Tier 4: Maintainer

**Definition**: Core team members responsible for project direction and maintenance.

**Requirements**:
- 50+ merged PRs or 100+ accepted templates
- 12+ months of active contribution
- Demonstrated leadership
- Voted in by existing maintainers

**Recognition**:
- Listed in README as maintainer
- Profile on project website
- Interview opportunities

**Access**:
- Full repository access
- Publish npm packages
- Manage releases
- Admin access to project resources

**Responsibilities**:
- Set technical direction
- Make architectural decisions
- Resolve conflicts
- Ensure code quality
- Represent project externally

---

### Tier 5: Advisory Board

**Definition**: External experts providing strategic guidance.

**Requirements**:
- Domain expertise in radiology/healthcare
- Nominated and voted in
- Time commitment to quarterly meetings

**Recognition**:
- Listed in GOVERNANCE.md
- Advisory board page
- Public acknowledgment

**Access**:
- Strategic input on roadmap
- Input on major decisions
- Access to private planning discussions

**Responsibilities**:
- Quarterly advisory meetings
- Provide strategic guidance
- Represent community perspective

## Promotion Process

### From Contributor to Regular Contributor
- Automatic based on contribution count
- Maintainer review for template contributions

### From Regular Contributor to Committer
- Sponsor from existing committer
- Maintainer approval
- 2-week review period

### From Committer to Maintainer
- Nomination by existing maintainer
- Advisory board review
- Full maintainer vote (75% required)

## Maintenance of Status

Contributors may be moved down in tier if:
- No activity for 12 months (except for health/military)
- Repeated violations of code of conduct
- Extended inactivity after promotion

Appeals can be made to the advisory board.
